## Відповіді на контрольні запитання

**1. Як визначається структура даних «множина»? Що таке хеш-таблиця? За якими критеріями можна класифікувати хеш-таблиці?**
Множина — це структура даних, що зберігає унікальні елементи, порядок яких не має значення. 
Хеш-таблиця — це структура даних для реалізації асоціативного масиву, яка використовує хеш-функцію для обчислення індексу в масиві, за яким зберігається або шукається елемент.
Хеш-таблиці класифікують за методами хешування та методами вирішення колізій (з відкритою адресацією або з роздільним зв'язуванням).

**2. Для чого призначено хешування? Які є методи хешування?**
Хешування призначене для перетворення ключа довільного розміру у значення фіксованого розміру (хеш-код), яке використовується як індекс масиву для надшвидкого доступу до даних $O(1)$. 
Методи: метод ділення (залишок від ділення ключа на розмір таблиці), метод множення, універсальне хешування тощо.

**3. За яких умов у хеш-таблиці виникають колізії? Якими способами їх вирішують?**
Колізії виникають, коли хеш-функція повертає однаковий індекс для двох різних ключів.
Методи вирішення: 
- *Роздільне зв'язування*: всі елементи з однаковим хешем зберігаються у зв'язному списку за цим індексом.
- *Відкрита адресація*: елемент розміщується у наступній вільній комірці масиву (шляхом лінійного, квадратичного зондування або подвійного хешування).

**4. Які вимоги висуваються до хеш-функцій?**
1. **Детермінованість:** для одного й того ж ключа функція завжди має повертати однаковий хеш.
2. **Швидкодія:** функція має обчислюватися швидко.
3. **Рівномірність:** хеш-значення повинні рівномірно розподілятися по всьому діапазону індексів таблиці для мінімізації кількості колізій.

**5. Що означає коефіцієнт заповнення хеш-таблиці та яку роль він відіграє?**
Коефіцієнт заповнення (load factor) — це відношення кількості збережених у таблиці елементів до її загального розміру. Він показує ступінь заповненості таблиці. Що вищий коефіцієнт, то більша ймовірність виникнення колізій і падіння продуктивності. При досягненні певного порогу таблицю зазвичай розширюють (рехешують).

**6. Які особливості реалізації хеш-таблиць з роздільним зв’язуванням, їх переваги і недоліки?**
Кожна комірка масиву містить вказівник на зв'язний список елементів.
- *Переваги:* простота реалізації видалення, відсутність проблеми кластеризації, таблиця може зберігати більше елементів, ніж її фізичний розмір (коефіцієнт заповнення $> 1$).
- *Недоліки:* додаткові витрати пам'яті на вказівники у списку, потенційно нижча швидкість через погіршену локальність даних у кеші процесора.

**7. Які особливості реалізації хеш-таблиць з відкритою адресацією, їх переваги і недоліки?**
Усі елементи зберігаються безпосередньо в масиві хеш-таблиці.
- *Переваги:* немає витрат пам'яті на посилання (як у списках), краща локальність кешу, що забезпечує швидший доступ до пам'яті.
- *Недоліки:* виникнення феномену первинної та вторинної кластеризації (скупчення елементів), обмежена місткість (коефіцієнт заповнення $\le 1$), складність операції видалення (потребує встановлення спеціального маркера "видалено", щоб не розірвати ланцюг пошуку).
