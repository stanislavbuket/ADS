# Лабораторна робота 1.6

## Мета
Дослідження методів аналізу ефективності алгоритмів та набуття практичних навичок з емпіричного дослідження швидкодії алгоритмів залежно від обсягу та структурованості вхідних даних.

## Завдання (Варіант 4)
- **Рівень 1**: Реалізувати алгоритм **Сортування – Шелла (за Шеллом)** для одновимірного масиву. Визначити час виконання для $N = 100, 10000, 1000000$.
- **Рівень 2**: Реалізувати додатковий алгоритм для порівняння: **Сортування – Шелла (за Кнутом)**. Отримати час виконання обох алгоритмів для розмірів $N, N^2, N^3$ ($N = 100$).
- **Рівень 3**: Згенерувати набори даних розміром 10000 елементів у трьох послідовностях: відсортованій (найменший час), випадковій та обернено відсортованій (найбільший час). Виміряти та порівняти час виконання обох алгоритмів на цих наборах.

## Відповіді на контрольні запитання

**1. Для чого необхідно аналізувати алгоритми? Які методи застосовуються для аналізу алгоритмів?**
Аналіз алгоритмів необхідний для обґрунтованого вибору найбільш ефективного способу розв'язання задачі в умовах обмежених ресурсів (часу та пам'яті).
Застосовуються два основних методи:
- **Емпіричний (апостеріорний):** тестування реального коду на реальних даних з вимірюванням часу виконання (в секундах) та спожитої пам'яті.
- **Математичний (апріорний):** теоретичний аналіз асимптотичної складності алгоритму як математичної функції залежно від розміру вхідних даних.

**2. У чому полягає сутність емпіричного аналізу алгоритмів? Які умови його застосування?**
Емпіричний аналіз полягає в проведенні експериментів над програмою. Вона запускається з різними наборами даних, і результати (час виконання) фіксуються та усереднюються.
*Умови застосування:* наявність повністю реалізованого та коректного коду алгоритму, наявність репрезентативних тестових даних, використання ізольованого середовища (щоб інші процеси ОС не впливали на чистоту замірів часу).

**3. У чому полягає сутність математичного аналізу? Коли його застосовують?**
Математичний аналіз дозволяє оцінити ефективність алгоритму абстрактно, незалежно від потужності процесора, мови програмування та оптимізацій компілятора. Оцінка виражається як функція від розміру входу $N$. Його застосовують на етапі проектування та розробки архітектури ще до написання коду.

**4. Що розуміють під «рост-функцією»? Які є типи «рост-функцій»?**
Рост-функція описує швидкість зростання обсягу обчислень алгоритму у відповідь на зростання розміру вхідних даних $N$.
Основні типи (від найшвидших до найповільніших):
- $O(1)$ — константна
- $O(\log N)$ — логарифмічна
- $O(N)$ — лінійна
- $O(N \log N)$ — лінійно-логарифмічна
- $O(N^2)$ — квадратична
- $O(2^N)$ — експоненціальна
- $O(N!)$ — факторіальна

**5. Що означає «$\Theta$-нотація», «$O$-нотація» та «$\Omega$-нотація»? Як можна їх зобразити графічно?**
Ці нотації описують асимптотичні межі швидкості росту функції:
- **$O$-нотація (О-велике):** позначає верхню межу. Алгоритм не буде працювати гірше (довше) за цю межу. Описує найгірший випадок виконання. Графічно крива $f(n)$ завжди лежить під кривою $c \cdot g(n)$ після певного $n_0$.
- **$\Omega$-нотація (Омега):** позначає нижню межу. Алгоритм завжди потребуватиме щонайменше цього часу. Описує найкращий випадок. Графічно крива $f(n)$ завжди лежить над $c \cdot g(n)$.
- **$\Theta$-нотація (Тета):** позначає точну межу (коли верхня і нижня межі збігаються з точністю до констант). Крива затиснута між двома графіками $c_1 g(n)$ та $c_2 g(n)$.

**6. Як застосовується математичний аналіз для оцінювання ефективності алгоритмів?**
Аналіз виконується шляхом:
1. Визначення базової операції алгоритму (наприклад, порівняння двох елементів).
2. Підрахунку кількості виконань цієї операції залежно від $N$.
3. Відкидання констант і доданків нижчого порядку, оскільки для великих $N$ вирішальне значення має тільки член, який зростає найшвидше (наприклад, $3N^2 + 5N + 10$ зводиться до $O(N^2)$).
